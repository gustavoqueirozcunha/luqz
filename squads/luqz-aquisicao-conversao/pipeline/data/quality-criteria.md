# Quality Criteria — Squad de Produto LUQZ

## Critérios Globais (aplicam-se a todos os ativos)

| Critério | Padrão de Aprovação |
|---|---|
| Objetivo de conversão | Todo ativo tem micro e macro-conversão definidos |
| Rastreabilidade ao ICP | Cada ativo foi criado para um ICP específico e identificável |
| CTA específico | O CTA diz o que acontece após o clique — nunca "saiba mais" ou "clique aqui" |
| Prova de afirmação | Toda promessa de benefício acompanhada de dado, depoimento ou garantia |
| Tom de voz | Consistente em todos os ativos do pacote |
| Handoff documentado | Cada agente documenta intenção de suas escolhas para que o próximo entenda |

---

## Critérios por Tipo de Ativo

### Estratégia de Performance
- [ ] Gargalo identificado com especificidade (dado ou observação, não suposição)
- [ ] Cada ativo listado tem função, etapa do ICP e métrica de sucesso
- [ ] Sequência de produção justificada por lógica de desbloqueio
- [ ] Benchmarks de referência do setor incluídos
- [ ] Riscos identificados com mitigação proposta

### Plano de Mídia
- [ ] CAC tolerável calculado com memória de cálculo
- [ ] Distribuição de verba em valor absoluto e porcentagem
- [ ] Canais excluídos têm justificativa
- [ ] Segmentação específica por canal (não genérica)
- [ ] Critérios de corte e escala numéricos definidos
- [ ] Cronograma semana a semana no mês 1

### Copy — Landing Page
- [ ] Headline em 3 versões com escolha justificada
- [ ] Hero: headline + subheadline + CTA primário
- [ ] Seção de problema amplifica dor sem culpar o leitor
- [ ] Benefícios listados como resultados, não como features
- [ ] Prova: mínimo 1 depoimento com resultado quantificado
- [ ] CTA final com contexto de baixo atrito
- [ ] Intenção de cada seção documentada

### Copy — Carrossel
- [ ] Slide 1 tem gancho de identificação ou curiosidade imediata
- [ ] Cada slide: máximo 40 palavras
- [ ] Progressão lógica entre slides (problema → agravamento → solução → prova → CTA)
- [ ] CTA no último slide: único, específico, baixo atrito
- [ ] Sugestão de visual incluída por slide

### Copy — Estático
- [ ] Headline: máximo 8 palavras visíveis, impacto imediato
- [ ] CTA: máximo 5 palavras, ação específica
- [ ] Descrição do visual complementar ao texto (não redundante)
- [ ] Total de palavras visíveis: máximo 15

### Roteiro de Vídeo
- [ ] Gancho em 3 versões, escolha justificada, posicionado nos primeiros 3 segundos
- [ ] Tempo estimado anotado em cada cena
- [ ] Tom emocional documentado em cada cena
- [ ] Visual na tela especificado em cada cena
- [ ] Copy escrita para ser falada (frases curtas, contrações naturais)
- [ ] CTA único, específico e posicionado no momento correto
- [ ] Duração total compatível com o formato (Reel: até 90s; VSL: 2–15min)

## Escala de Avaliação

| Nota | Significado |
|---|---|
| 5 | Padrão de referência — pode ser usado como template interno |
| 4 | Aprovado — cumpre todos os critérios obrigatórios |
| 3 | Aprovado com ressalvas — cumpre critérios, mas tem melhorias relevantes |
| 2 | Requer revisão — problema que não bloqueia entrega, mas compromete resultado |
| 1 | Rejeitado — problema crítico que compromete o objetivo de conversão |

---

## Critérios de Coerência de Pacote

Antes da aprovação final, verificar:

- [ ] Tom de voz consistente em todos os ativos (mesma persona, mesmo registro)
- [ ] ICP idêntico — todos os ativos falam com o mesmo perfil de cliente
- [ ] Mensagem central coerente — a promessa é a mesma em todos os pontos de contato
- [ ] Formatos do plano de mídia existem na copy produzida
- [ ] Nenhum ativo sem função definida na jornada do ICP
