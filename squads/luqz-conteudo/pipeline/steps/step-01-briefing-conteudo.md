# Step 01: Briefing de Conteúdo

## Instrução ao Usuário

Preencha o modelo abaixo. Todos os campos (exceto Referência) são obrigatórios. Os próximos steps dependem exclusivamente das informações fornecidas aqui.

## Modelo de Input

```markdown
# Briefing de Conteúdo

**Nicho:** [ex: contabilidade para MEI / SaaS B2B / e-commerce de moda]
**ICP:** [descreva quem é o cliente ideal — cargo, faixa etária, dor principal, nível de consciência]
**Objetivo:** [o que esse conteúdo deve provocar — gerar leads, construir autoridade, converter, educar]
**Produto/Serviço:** [o que está sendo vendido ou promovido]
**Tom de Voz:** [ex: consultivo, direto, provocador, educativo, informal]
**Estágio do Funil:** [consciência / consideração / decisão]
**Referência:** [opcional — @conta, link de carrossel, estilo visual de referência]
```

## Regras do Checkpoint

- O briefing deve ser preenchido pelo usuário antes de qualquer step avançar
- Nenhum campo obrigatório pode ficar em branco
- O campo "Referência" pode conter "sem referência" se não houver
- O output salvo em `briefing-conteudo.md` é a única fonte de verdade para todos os steps seguintes

## Quality Criteria

- [ ] Nicho definido com especificidade (não genérico como "marketing")
- [ ] ICP com pelo menos: cargo/perfil, dor principal e nível de consciência
- [ ] Objetivo com verbo de ação claro (gerar, converter, educar, etc.)
- [ ] Tom de voz nomeado (ao menos uma palavra-chave)
- [ ] Estágio do funil selecionado (consciência / consideração / decisão)
